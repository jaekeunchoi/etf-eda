# Streamlit 대시보드 복구 및 차트 개편 계획

사용자 요청에 따라 데이터의 실시간 조회 및 소스코드 수정이 즉각 반영되는 **Streamlit 대시보드(`src/app.py`)** 환경으로 복구합니다. 이와 함께 대시보드 내의 운용사 점유율 도넛 차트를 **막대 그래프**로 변경하고, **운용사별 트리맵**을 고도화하여 시가총액 분포를 더 세련되게 시각화합니다.

## 사용자 검토 요구사항
> [!NOTE]
> * **서버 전환**: 기존의 정적 index.html 서버(포트 8000)를 중단하고, 실시간 데이터 새로고침 및 코드 수정이 즉각 반영되는 Streamlit 서버(포트 8501)를 구동합니다.
> * **차트 개편**: 운용사별 시장 점유율을 표현하는 도넛 차트를 막대 차트로 변경하며, 운용사-종목 계층의 트리맵을 전면에 배치하여 시가총액 지배력을 직관적으로 파악할 수 있게 합니다.

## 제안된 변경 사항

---

### 1. Streamlit 대시보드 차트 개편
#### [MODIFY] [app.py](file:///c:/Users/student/Documents/etf-eda/src/app.py)
* **도넛 차트 -> 막대 차트 변경**: 운용사별 시가총액 점유율(%)을 가독성이 높은 가로/세로 막대 그래프(`px.bar`)로 시각화합니다.
* **운용사별 트리맵 고도화**: 운용사 및 소속 ETF 종목들이 시장 시가총액에서 차지하는 비중을 면적으로 표현하는 트리맵(`px.treemap`)을 더 가독성 있고 화려한 컬러 스케일(예: 시총 규모별 그라데이션)로 개선합니다.

---

### 2. 서버 환경 전환
* **로컬 웹 서버 교체**: 
  * 현재 구동 중인 Python `http.server` (포트 8000)를 종료합니다.
  * `streamlit run src/app.py` 명령을 실행하여 로컬 포트 8501에서 대시보드를 재구동합니다.

## 검증 계획

### 수동 검증 및 동작 테스트
1. **Streamlit 서버 구동 검증**:
   * 로컬에서 `streamlit run src/app.py`를 구동하여 에러 없이 서버가 열리는지 확인합니다.
2. **차트 개편 검증**:
   * 브라우저로 대시보드(http://localhost:8501)에 접속하여 운용사별 점유율 막대 그래프와 종목별 시총 트리맵이 정상적으로 출력되는지 확인합니다.
3. **실시간 데이터 새로고침 검증**:
   * 사이드바의 '실시간 데이터 새로고침' 버튼 클릭 시 실시간 API 호출을 통해 최신 데이터가 반영되는지 테스트합니다.
