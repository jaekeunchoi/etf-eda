# GitHub Actions 1분 주기 데이터 수집 자동화 계획

사용자 요청에 따라 네이버 금융 ETF 데이터를 1분 주기로 실시간 수집하여 해당 깃허브 저장소의 `data/` 폴더 내부에 자동으로 저장·커밋·푸시하는 파이프라인을 구축합니다. 

## 사용자 검토 요구사항
> [!WARNING]
> **GitHub Actions의 Cron 스케줄 제약사항**
> * GitHub Actions 공식 Cron 스케줄러의 최소 실행 간격 제한은 **5분**입니다. (`* * * * *`와 같이 1분으로 설정하더라도 실제로는 5분 이상 간격으로 실행됩니다.)
> * 이를 우회하여 **실제 1분 주기**에 가깝게 동작하도록 아래의 두 가지 방안을 제안합니다.

---

## 제안된 변경 사항

### 방안 A. (추천) 5분 주기 공식 스케줄러 실행
* **동작 방식**: `.github/workflows/update_data.yml`을 5분 간격(`cron: "*/5 * * * *"`)으로 실행합니다.
* **장점**: GitHub Actions 실행 시간이 최소화되어 월 무료 크레딧 소모가 적고, 깃허브 계정이 스패밍으로 제한될 위험이 없습니다.

### 방안 B. 1분 루프 우회 실행 (Actions 내부 Sleep 활용)
* **동작 방식**: 5분 간격(`cron: "*/5 * * * *"`)으로 Actions가 실행되면, 가상환경 내부에서 1분 간격으로 루프를 돌며 총 5회 수집(약 4분 40초 동안)을 수행하고, 수집된 데이터를 최종 커밋 및 푸시한 후 종료합니다.
* **단점**: GitHub Actions가 24시간 동안 거의 끊임없이 러닝 상태를 유지하므로, 무료 크레딧(월 2,000분)이 약 14일 이내에 모두 소진됩니다.

---

### [NEW] 데이터 수집 주기 제어 워크플로우 구성
사용자 승인 후 선택하신 방안에 맞게 `.github/workflows/update_data.yml` 파일을 생성/업데이트합니다.

#### 1안: A안(5분 주기) 적용 시 `.github/workflows/update_data.yml`
```yaml
on:
  schedule:
    - cron: '*/5 * * * *'
```

#### 2안: B안(1분 루프 우회) 적용 시 `.github/workflows/update_data.yml`
```yaml
# 내부 파이썬 스크립트에 loop 및 interval 옵션을 활용하여 5분마다 실행된 워크플로우가 1분 단위 수집을 5회 수행 후 커밋
run: |
  python src/fetch_etf.py --loop --interval 60 --duration 260
```
*(참고: `fetch_etf.py`에 수집 실행 누적 시간제한 `--duration` 옵션을 지원하도록 파이썬 코드를 추가 보완할 수 있습니다.)*

## 검증 계획

### 자동 및 수동 검증
1. **GitHub Actions 수동 실행 테스트**:
   * 워크플로우를 수동 트리거하여 `data/` 폴더에 데이터 이력 및 최신 파일이 정상 커밋·푸시되는지 확인합니다.
2. **수집 주기 타임스탬프 검증**:
   * 깃 로그의 커밋 타임스탬프 또는 저장된 CSV/JSON 파일명 타임스탬프를 대조하여 주기가 정상 작동하는지 확인합니다.
