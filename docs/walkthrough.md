# 📊 실시간 ETF 종합 EDA 대시보드 구축 결과 보고서 (Walkthrough)

본 보고서는 네이버 금융 실시간 API와 Streamlit을 활용해 구축한 인터랙티브 ETF 대시보드 프로젝트의 변경 및 구현 내용, 검증 결과를 요약합니다.

---

## 🛠️ 주요 구현 내용

프로젝트의 규칙 및 설계 구조에 맞춰 코드를 패키지화하여 `src/` 폴더 하위에 격리하여 구현했습니다.

```

├── .venv/
├── docs/
│   ├── implementation_plan.md  # 구현 계획서 복사본
│   ├── task.md                 # 태스크 리스트 복사본
│   └── walkthrough.md          # [NEW] 본 결과 보고서 복사본
├── src/
│   ├── __init__.py          # [NEW] 패키지 초기화 파일
│   ├── data_loader.py       # [NEW] 실시간 API 수집 및 데이터 정제 모듈

└── fetch_etf.py             # [MODIFY] 구현 계획서 전처리 포맷이 적용된 CSV 저장 스크립트
```

### 1. 실시간 데이터 로더 구현 (`src/data_loader.py`)
- **실시간 API 연동**: 네이버 금융 ETF 리스트 API를 실시간 호출하도록 설계했습니다. JSONP 형태와 순수 JSON 형태를 모두 안전하게 추출하여 복구할 수 있는 파싱 헬퍼 함수(`extract_json_from_jsonp`)를 구축해 장애 대응력을 높였습니다.
- **데이터 한글 직관화 및 타입 포맷팅**: `itemname` -> `종목명` 등 금융 전문 용어에 부합하는 컬럼명 매핑 및 실수·정수형 전처리를 일괄 적용했습니다.
- **다양한 파생변수 생성**:
  - **NAV 괴리율(%)**: 현재 거래가와 순자산가치 간의 괴리 정도를 실시간 추적할 수 있도록 연산 컬럼 추가
  - **운용사 브랜드 분류**: 종목명의 접두어(KODEX, TIGER, ACE, KBSTAR 등)를 정형화하여 소속 자산운용사명을 자동 할당하는 파서 장착

### 2. 배치 수집 스크립트 리팩토링 및 실행 (`fetch_etf.py`)
- **데이터 전처리 표준 통합**: 신규 대시보드와 동일한 데이터 품질과 포맷을 유지하기 위해, `src.data_loader`의 `load_realtime_etf_data()` 가공 모듈을 활용하여 데이터를 로드하도록 구조를 전면 개선했습니다.
- **실시간 데이터 동적 저장**:
  - `etf_items_latest.csv` (최신 갱신 파일)와 `etf_items_YYYYMMDD_HHMMSS.csv` (이력 파일)의 두 가지 포맷으로 실시간 수집 가공 데이터를 지정된 `data/` 디렉토리에 동적으로 강제 덮어쓰기하여 저장하도록 설계했습니다.
  - 한글 인코딩 깨짐을 예방하기 위해 `utf-8-sig` 형식으로 표준화하여 작성 및 테스트를 마쳤습니다.

### 3. Streamlit 대시보드 애플리케이션 (`src/app.py`)
- **요약 KPI 메트릭스**: 총 상장 종목 수, 총 시가총액(조 원 단위 변환), 총 일 거래대금, 상승률/하락률 종목 비율 및 3개월 평균 수익률을 상단에 그리드로 가시성 있게 배치했습니다.
- **사이드바 대화형 필터**: 종목명/종목코드 즉시 검색창, 자산운용사 다중 선택 필터, 상승/하락 여부 라디오 버튼 및 시가총액 범위 조절 슬라이더를 유기적으로 배치했습니다.
- **다중 분석 탭(Tab) 구조 설계**:
  - **Tab 1: 시장 요약 & 운용사 점유율**: 운용사별 점유율 Treemap 및 3개월 성과 비교 Bar Chart 탑재
  - **Tab 2: 부문별 TOP 10 순위**: 시가총액, 거래대금, 등락률 상/하위 랭킹 시각화
  - **Tab 3: 분포 및 관계 (EDA)**: 3개월 수익률 히스토그램, 시가총액-거래대금 관계 Scatter Plot, 수치형 변수 상관관계 Heatmap, 운용사별 괴리율 분포 Box Plot 설계
  - **Tab 4: 실시간 데이터 그리드**: 검색·필터링된 표 제공 및 CSV 포맷 메모리 내보내기 기능 추가
  - **Tab 5: 개별 종목 정밀 진단**: 단일 ETF를 선택하면 20년차 데이터 분석가 관점의 리스크, 유동성, 가격 괴리율 평가 처방전을 자동으로 동적 구성

### 4. Git 버전 관리 제외 설정 (`.gitignore`)
- **버전 관리 불필요 파일 격리**: 프로젝트 내 파이썬 가상환경(`.venv`), 컴파일된 캐시 파일(`__pycache__/`, `*.pyc`), 데이터 디렉토리 내 임시 수집 데이터 파일(`data/*`), 그리고 IDE 설정 및 OS 시스템 파일 등을 관리 대상에서 원천 배제하여 원격 저장소 용량 낭비 및 충돌을 방지하도록 `.gitignore` 파일을 작성했습니다.

## 🔍 검증 결과 (Verification Results)

### 1. 데이터 로더 단위 테스트
파이썬 가상환경에서 `data_loader.py` 모듈을 단독 실행하여 데이터 스키마 가공을 확인했습니다.
- **결과**: `1150+` 개의 실시간 종목 정보를 정상 수집하였으며, 한글 컬럼 및 `NAV괴리율(%)`과 `운용사` 파생 변수가 유실 없이 추가된 것을 정상 확인했습니다.

### 2. Streamlit 웹 애플리케이션 실행 테스트
- **실행 명령**: `uv run streamlit run src/app.py --server.port 8502 --server.headless true`
- **실행 결과**:
  - 로컬 웹 서버가 `http://localhost:8502`에서 에러 없이 기동되었습니다.
  - 외부 브라우저 테스트 서브에이전트 실행 시, Playwright 드라이버 다운로드 주소(404 에러) 문제로 캡처에는 실패하였으나 실제 서버 프로세스는 데몬 형태로 오류 없이 실행되어 서비스 제공 대기 상태인 것을 수동 로그 조회를 통해 최종 검증했습니다.
